Recommended Architecture

A modular, Python-centric architecture with SSH orchestration is ideal. It’s lightweight, leverages Chia’s Python foundation, works seamlessly across Windows 11 (controller) and Ubuntu (harvesters), and scales well from scripts to a full GUI application.

1 Core Architecture: Controller-Agent Pattern

Central Controller

  • • Runs on your Windows 11 machine
  • • Manages configuration, UI, and orchestration
  • • Monitors deployment progress

Harvesters (6 Ubuntu Servers)

  • • Agentless design (SSH only)
  • • Key-based SSH authentication
  • • Local network only

Technology Stack

Core Language

Python 3

Cross-platform, excellent SSH libraries, and native to Chia.

Phase 1: Scripting

  • Paramiko or Fabric — SSH connections
  • asyncio / threading — Parallel deployments
  • YAML/JSON — Configuration

Phase 2: GUI

  • CustomTkinter or PyQt/PySide6 (Recommended)
  • Individual harvester monitoring panels
  • Real-time status & logs

Optional Enhancements

  • Ansible — For robust, idempotent deployments
  • SQLite — Harvester inventory & history
  • keyring — Secure credential storage

Implementation Phases

PHASE 1

Automated Deployment Script

CLI tool to deploy from your Windows machine via SSH.

  • • Stop harvester services
  • • Backup configuration
  • • Update Chia (git or installer)
  • • Restart and verify
  • • Real-time progress + summary report
PHASE 2

Full GUI Application

Dashboard with individual monitored displays for each harvester.

Harvester cards with status
Bulk + individual deploy
Live logs & progress

Security & Best Practices

  • ✓ SSH key authentication only
  • ✓ Limit concurrent deployments
  • ✓ Automatic backups before upgrade
  • ✓ Error handling & rollback
  • ✓ Local network firewall rules
  • ✓ Version control your code
